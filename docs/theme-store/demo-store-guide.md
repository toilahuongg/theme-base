# Demo Store Guide

Theme Store demos should show how the theme works in a real store, not a synthetic showcase with unrealistic assumptions.

## Match The Demo To The Theme

- Use an industry that matches the blueprint.
- Use a catalog size that matches the intended preset.
- Keep the visual style consistent with the documented positioning.
- Make the navigation and collections believable for the products being sold.

## Copy And Imagery

- Use authentic merchant-style copy, not placeholder marketing language.
- Use licensed imagery only.
- Avoid claiming features that the theme does not actually provide.
- Do not present app-powered behavior as if it is built into the theme.

## Payments And Checkout

- Use test payments where appropriate.
- Make sure the demo flow can be exercised without breaking checkout expectations.
- Verify product pages, cart behavior, and key conversion paths before recording or sharing the demo.

## Demo Hygiene

- Keep product data clean and coherent.
- Keep pricing, variants, and collections internally consistent.
- Make sure any featured content reflects the actual theme configuration.
- Recheck the demo after regeneration so source changes are visible in the preview.

## Demo Rule

The demo store should prove the theme's intended use case, not hide gaps with fake functionality.
