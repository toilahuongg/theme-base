# Catalog Strategies

Theme Base blueprints should reflect the merchant's real catalog size. Different catalog sizes need different navigation, merchandising, and template emphasis.

## 1 Product

Use this when the merchant sells one hero product or a single flagship item.

- Make the product page carry most of the selling work.
- Emphasize trust, benefits, social proof, and clear calls to action.
- Keep navigation simple and avoid unnecessary collection complexity.
- Treat bundles, FAQs, and comparison blocks as first-class content.

## Few (2-10)

Use this when the store has a small, curated range.

- Highlight product differences clearly.
- Add comparison-oriented content.
- Use editorial home content to frame the range.
- Keep collection navigation shallow and obvious.

## Some (11-100+)

Use this for a moderate catalog with meaningful browsing needs.

- Make collection discovery a central pattern.
- Use filters, collection cards, and editorial category entry points.
- Keep product hierarchy clear so shoppers can move between product groups.
- Balance merchandising with search and browse support.

## Lots (500+)

Use this for large catalogs that need scale and efficiency.

- Prioritize search, filtering, and faceted navigation.
- Keep category architecture disciplined.
- Use quick add and fast comparison patterns where appropriate.
- Optimize performance and avoid heavy page structures that slow browsing.

## Authoring Rule

Blueprints should state the catalog strategy explicitly so downstream sections, templates, and content defaults can be generated for the right shopping behavior.
